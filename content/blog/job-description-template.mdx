A maioria das descrições de vaga é genérica e chata ("Empresa líder de mercado busca proativo..."). Os melhores talentos ignoram isso.
Uma boa JD é uma **carta de vendas**.

## Estrutura Vendedora
1.  **A Missão (O Porquê):** O que estamos construindo? "Ajude a levar internet para Marte."
2.  **O Desafio (O Quê):** O que você vai fazer nos primeiros 6 meses? "Refatorar o sistema de pagamentos legado." (Seja específico, devs amam desafios).
3.  **A Autonomia (O Como):** Que ferramentas vai usar? Quem decide a stack?
4.  **A Recompensa (WIIFM):** Salário, Equity, Benefícios reais (não "mesa de ping pong").

## O que evitar
*   Listas infinitas de requisitos (ex: "10 anos de Swift" sendo que Swift tem 5 anos).
*   Palavras vazias: "rockstar", "ninja", "vestir a camisa".

## Exemplo de JD (Trecho)
> **Sua Missão:** Reduzir o tempo de build de 20min para 5min.
> **Stack:** Rust, Go, AWS.
> **Salário:** R$ 15k - 20k CLT.

## Quer gerar uma JD incrível?
Use a **Mesa Carreira** ou **Mesa Produto**. Diga "Preciso contratar um Senior React Dev" e a IA vai gerar uma JD focada em atrair esse perfil específico.
